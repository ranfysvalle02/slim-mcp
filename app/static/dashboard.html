It was never the protocol

Same MCP, same model, same task.
A fraction of the bill.

MCP just hands a model its tools. Whether that handoff costs twenty thousand tokens or a few hundred is decided above the wire — never by the protocol.

A kitchen-sink server ships the whole binder every turn. A smart MCP gateway with MongoDB embeds the binder once, then — when you just say what you're doing — retrieves only the tools that task needs by meaning (MongoDB $vectorSearch). The tools it hands back are byte-for-byte identical to the firehose — the only thing that changes is how many. Nothing to declare, no key, no cloud: the model runs on your own hardware.

The one rule it does hardcode: a safety floor that blocks destructive tools (like delete_file) in both directions — never listed, never callable. See how it works and how to edit the blocklist below.

Add / edit the blocklist. The needles live in app/gateway/github_proxy.py (the DESTRUCTIVE_NEEDLES tuple) — add a substring like "remove" or "force" to hide more. Two clients. One difference: which server you point at.

Same library, same model, same task. The raw client talks to the official GitHub MCP server and gets the whole binder; the smart client talks to the smart MCP gateway (MongoDB-backed) and just says what it's doing via an x-mcp-query header. Nothing to declare up front.

RAW
# Official GitHub MCP — the whole binder
from fastmcp import Client

client = Client(
  "https://api.githubcopilot.com/mcp/x/all",
  headers={"Authorization": "Bearer $GITHUB_PAT"},
)
async with client:
  tools = await client.list_tools()
  # -> the whole catalog, full schemas
SMART
# Smart MCP gateway (MongoDB) — only what the task needs
from fastmcp import Client

client = Client(
  "http://localhost:8000/mcp/",
  headers={"x-mcp-query": "review this pull request"},
)
async with client:
  tools = await client.list_tools()
  # -> only the tools this task needs — same full schemas, far fewer tokens

The gateway embeds the whole catalog once. The x-mcp-query is embedded and matched against it with a single MongoDB $vectorSearch, and only the top-k tools — ranked by meaning — come back. With no query header it behaves like an honest proxy and returns the full catalog.
